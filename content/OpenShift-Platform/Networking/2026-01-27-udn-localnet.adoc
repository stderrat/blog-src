This blog post summerizes our learings around using User Defined
Networking (UDN) with the localnet topology. The goal is to deploy a
pod with a secondary network interface in a specific VLAN. The pod
should be reachable from outside the cluster.

<!--more-->

OVNKubernetes supports adding secondary networks (there could be more
than one secondary network) to pods. This is also applicable to
Virtual Machines hosted on OpenShift Virtualization (KubeVirt).

There are three possible network topologies:

- Layer 2: Basically a VLAN spanned over all nodes in the
  cluster. This is per default not reachable from outside the
  cluster. UDN BPG could be used to expose such a configuration.
- Layer 3: A layer 3 IP subnet per node similar to the pod network, interconnected via
  routers. Again not reachable from the outside, UDN BPG could be used
  to announce this network to routes outside of OCP.
- Localnet: A Layer 2 topology with a connection to a
  physical network. In our case the Localnet layer 2 bridge is
  connected to br-ex.
