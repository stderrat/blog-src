This blog post summerizes our learings around using User Defined
Networking (UDN) with the localnet topology. The goal is to deploy a
pod with a secondary network interface in a specific VLAN. The pod
should be reachable from outside the cluster.

<!--more-->

OVNKubernetes supports adding secondary networks (there could be more
than one secondary network) to pods. This is also applicable to
Virtual Machines hosted on OpenShift Virtualization (KubeVirt).

There are three possible network topologies:

- Layer 3: A layer 3 IP subnet per node similar to the pod network,
  interconnected via OVN virtual routers and a OVN virtual switch per
  node. The Layer 3 subnets are not reachable from the outside, UDN
  BPG could be used to announce this network to routes outside of OCP.
- Layer 2: Basically a VLAN spanned via a OVN virtual switch over all
  nodes in the cluster. Again this is per default not reachable from
  outside the cluster. UDN BPG could be used to expose such a
  configuration.
- Localnet: A Layer 2 topology with a connection to a physical
  network. There is a patch port from the virtual switch to
  br-ex. br-ex is a Open vSwitch bridge where the interface of a node is

== Creating a localnet topology configuration


We would like to explore two separate scenarios:

1. A simple connection for VM's to the outside world. This connects a
virtual machine to the same physical network as the node where it is
running. The node is on an access port.

2. A more involved configuration where the VM is located in a separate
VLAN. Our OpenShift node has a trunk port configured and the virtual
machine should be only reachable in a dedicated VLAN.

For more information about VLANS see https://en.wikipedia.org/wiki/VLAN.

Lets take a look at our current OVN configuration.

=== Simple VM connectivity

First let's create a namespace for testing out *localnet* configuration:

[source,yaml]
----------
apiVersion: v1
kind: Namespace
metadata:
  labels:
    environment: "udn-localnet"
  name: test-udn-localnet
----------

Next we create our _ClusterUserDefinedNetwork_.

[source,yaml]
----------
apiVersion: k8s.ovn.org/v1
kind: ClusterUserDefinedNetwork
metadata:
  name: cudn-localnet
spec:
  namespaceSelector:
    matchExpressions:
    - key: kubernetes.io/metadata.name
      operator: In
      values:
      - test-udn-localnet <1>
  network:
    localnet:
      ipam:
        mode: Disabled
      physicalNetworkName: localnet1
      role: Secondary
    topology: Localnet
----------
<1> This matches the namespaces we created above
